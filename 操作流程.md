# 線上補簽同意書操作流程

本文件說明 Signature-onweb 第一版的實際操作流程，包含老師端、家長端與系統暫存清除流程。

## 一、使用前準備

系統設計者需先完成：

1. 建立 Google Drive 暫存資料夾。
2. 取得 Google Drive folder ID。
3. 在 Google Cloud 啟用 Google Drive API。
4. 建立 OAuth Desktop app。
5. 取得 OAuth client ID、client secret 與 refresh token。
6. 將以下資料填入本機 `.env` 或部署平台 Secrets：

```env
GOOGLE_AUTH_MODE=oauth
GOOGLE_OAUTH_CLIENT_ID=
GOOGLE_OAUTH_CLIENT_SECRET=
GOOGLE_OAUTH_REFRESH_TOKEN=
GOOGLE_DRIVE_TEMP_FOLDER_ID=
TEMP_CASE_TTL_HOURS=4
MAX_UPLOAD_SIZE_MB=5
PORT=3000
```

## 二、老師端流程

### 1. 建立同意書

1. 老師開啟首頁。
2. 老師上傳圖片或 PDF 同意書。
3. 檔案上限為 5MB。
4. PDF 第一版只取第 1 頁作為簽名底圖。
5. 系統會使用上傳檔名作為案件標題。

### 2. 設定簽名欄

1. 點選「新增簽名欄」。
2. 系統新增一個預設文字為「家長簽名」的欄位。
3. 老師可拖曳簽名欄移動位置。
4. 老師可拖曳右下角黃色縮放把手調整大小。
5. 欄位座標與大小會依文件比例記錄，因此家長在不同手機上開啟也能對齊。

### 3. 下載空白預覽

1. 老師點選「下載空白簽名欄預覽圖」。
2. 系統輸出 PNG 圖片。
3. 圖片包含同意書底圖與尚未簽名的簽名欄。

### 4. 產生家長連結

1. 老師點選「產生家長連結」。
2. 系統將同意書檔案暫存到 Google Drive。
3. 系統建立一筆案件設定，包含：
   - 案件 ID
   - 同意書標題
   - Google Drive 檔案 ID
   - 簽名欄座標
   - 到期時間
4. 系統產生家長專用連結。
5. 第一版一份連結只給一位家長使用。

## 三、家長端流程

### 1. 開啟連結

1. 家長收到老師提供的連結。
2. 家長可用手機、平板或電腦開啟。
3. 家長不需要登入。
4. 系統依案件 ID 讀取同意書與簽名欄設定。
5. 若案件已完成、已刪除或超過 4 小時，系統顯示連結失效。

### 2. 手寫簽名

1. 家長在手寫板簽名。
2. 家長可清除重簽。
3. 家長點選「套用簽名」。
4. 系統將簽名圖片放入指定簽名欄。

### 3. 下載完成圖片

1. 家長點選「下載完成圖片」。
2. 系統用 Canvas 合成：
   - 原始同意書
   - 老師設定的簽名位置
   - 家長手寫簽名
3. 家長下載 PNG 圖片。
4. 若手機瀏覽器無法自動下載，畫面會顯示完成圖片，可長按保存。
5. 系統通知後端案件已完成。
6. 後端刪除 Google Drive 暫存檔案與案件設定。

## 四、系統清除流程

### 完成後清除

1. 家長下載完成圖片後，前端呼叫完成 API。
2. 後端刪除 Google Drive 暫存檔案。
3. 後端刪除案件設定。
4. 原家長連結失效。

### 過期清除

1. 系統定期掃描暫存案件。
2. 找出超過 4 小時仍未完成的案件。
3. 刪除 Google Drive 暫存檔案。
4. 刪除案件設定。

## 五、例外狀況

### 檔案超過 5MB

系統顯示：

```text
檔案超過 5MB，請壓縮後再上傳。
```

### 檔案格式不支援

第一版只支援：

- 圖片
- PDF

### 家長連結失效

可能原因：

- 案件已完成。
- 案件已超過 4 小時。
- 暫存檔案已被清除。
- 連結中的案件 ID 錯誤。

### Google Drive 寫入失敗

可能原因：

- `GOOGLE_OAUTH_REFRESH_TOKEN` 未設定。
- Google Drive API 未啟用。
- OAuth refresh token 已失效。
- `GOOGLE_DRIVE_TEMP_FOLDER_ID` 錯誤。

## 六、目前驗收狀態

已完成並測試：

- OAuth 模式寫入 Google Drive。
- 從 Google Drive 讀回同意書檔案。
- 完成後刪除 Drive 暫存檔。
- 家長連結完成後失效。
- `.env` 與 `.tmp/` 不會上傳 GitHub。

## 七、發布後必須修改

<div style="background:#000;color:#fff;padding:16px;border-radius:8px;">

如果要把 GitHub 專案公開，或部署成家長手機可使用的正式服務，請務必完成以下修改：

1. 確認 GitHub 只包含程式碼、README、操作流程與 `.env.example`。
2. 確認 `.env` 沒有被提交到 GitHub。
3. 將 OAuth client ID、client secret、refresh token 放到部署平台 Secrets，不要寫在程式碼裡。
4. 正式使用前建議重新建立 OAuth client secret 與 refresh token。
5. 將 `GOOGLE_DRIVE_TEMP_FOLDER_ID` 設成正式暫存資料夾。
6. 部署完成後，使用正式公開網址測試老師端與家長端，不要再用 `localhost` 給家長。
7. 若 Google OAuth 仍是 Testing 模式，只有測試使用者可以授權；正式使用前需確認測試名單或發布狀態。
8. 若學校或大量家長使用，建議另行建立專用 Google 帳號與專用 Drive 暫存資料夾。

## 八、遠端與本機端發布

### 遠端 Web

家長手機要在異地開啟連結時，必須部署到公開網址。可使用 Render、Railway、Cloud Run 或其他 Node.js 主機。

本專案已提供：

- `Dockerfile`
- `render.yaml`

遠端部署時，請將 OAuth 與 Google Drive 參數放到部署平台 Secrets，不要放進 GitHub repo。

### 本機 Windows exe

本機端可建立 Windows 啟動器：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-windows-exe.ps1
```

輸出：

```text
dist/Signature-onweb-windows.zip
```

使用者解壓縮後，執行：

```text
Signature-onweb.exe
```

exe 會啟動本機服務並開啟瀏覽器。若要寫入 Google Drive，解壓縮後仍需在同一資料夾建立 `.env`。
