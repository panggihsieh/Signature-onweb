# Signature-onweb 線上補簽同意書

Signature-onweb 第二版改成純前端排版連結流程。

老師先上傳同意書、排好簽名欄位置，再把「排版好的網址」和「同意書原檔」一起傳給家長。家長開啟網址後，自行貼上或上傳老師傳來的同意書，完成手寫簽名後下載 PNG。

第二版不再依賴 Google Drive、不建立後端暫存案件，也不需要家長登入。

## 第二版重點

- 老師端可上傳圖片或 PDF 同意書。
- 上傳檔案大小限制 5MB。
- PDF 目前仍使用第 1 頁作為簽名底圖。
- 老師可新增、移動、縮放簽名欄。
- 老師可下載空白簽名欄預覽圖。
- 老師可產生只包含排版資訊的家長連結。
- 家長可上傳或直接貼上老師傳來的同意書圖片。
- 家長可在手機或電腦手寫簽名。
- 家長簽完後可下載 PNG 完成圖。

## 第二版流程

```text
老師上傳同意書
  -> 老師調整簽名欄
  -> 系統產生排版連結
  -> 老師把排版連結 + 同意書原檔傳給家長
  -> 家長開啟排版連結
  -> 家長貼上或上傳同意書
  -> 家長手寫簽名
  -> 下載 PNG
```

## 和第一版的差異

- 不再把同意書上傳到 Google Drive。
- 不再建立後端案件 ID 與過期清除流程。
- 連結只保存簽名欄座標、尺寸、標題等排版資料。
- 家長必須另外取得老師傳來的同意書原檔。
- 只要老師與家長使用的是同一份文件，簽名欄就能對齊。

## 使用限制

- 家長端貼上的文件版面比例，必須和老師排版時接近。
- 若家長使用不同截圖、被裁切過的圖片、不同頁面的 PDF，簽名欄可能偏移。
- 目前仍只輸出 PNG，不直接回存 PDF。
- PDF 家長端建議直接用上傳，不建議期待系統支援從剪貼簿直接貼 PDF。

## 本機啟動

```bash
node server.js
```

開啟：

```text
http://localhost:3000
```

語法檢查：

```bash
node --check server.js
node --check app.js
```

目前這個專案使用 Node.js 內建 HTTP server，不需要額外安裝 npm 套件。

## 遠端部署

如果老師要把排版連結傳給異地家長，服務仍然必須部署到公開網址，不能只跑在 `localhost`。

本專案已提供：

- `Dockerfile`
- `render.yaml`

部署到 Render 時，系統可直接使用公開網址產生排版連結。若之後改用自訂網域，可設定：

```env
PUBLIC_BASE_URL=https://your-domain.example
```

## 目前保留的環境變數

第二版實際會用到的只有：

```env
PUBLIC_BASE_URL=
PORT=3000
```

`.env.example` 內仍保留第一版 Google 相關欄位，原因是未來若要回頭做混合模式或第三版擴充，可以沿用既有部署結構；但第二版流程本身不需要填那些值。

## 第二版驗收標準

- 老師可上傳 5MB 以內的圖片或 PDF。
- 老師可拖曳並縮放簽名欄。
- 老師可下載空白簽名欄預覽圖。
- 老師可產生排版好的家長連結。
- 家長可開啟該連結並看到預設簽名欄。
- 家長可貼上或上傳老師傳來的同意書。
- 家長可手寫簽名並套用到文件。
- 家長可下載含簽名的 PNG。
