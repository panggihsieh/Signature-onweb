# 第二版工作流程

本文件描述 Signature-onweb 第二版的產品流程與系統邏輯。第二版目標是降低部署與儲存複雜度，只保留老師排版、家長簽名、下載 PNG 這條最短主流程。

## 一、設計原則

1. 不使用 Google Drive。
2. 不依賴後端暫存案件。
3. 不要求家長登入。
4. 老師只負責排版簽名欄與分享連結。
5. 家長自行載入老師傳來的同意書原檔。
6. 最終輸出固定為 PNG。

## 二、角色分工

### 老師

1. 上傳同意書。
2. 調整簽名欄。
3. 產生排版連結。
4. 把排版連結和同意書原檔傳給家長。

### 家長

1. 開啟老師提供的排版連結。
2. 貼上或上傳老師傳來的同意書。
3. 手寫簽名。
4. 下載完成 PNG。

### 系統

1. 記錄老師設定的簽名欄位置與尺寸。
2. 將排版資料編碼到網址。
3. 在家長端讀回排版資料。
4. 將簽名套用到家長提供的文件。
5. 產出 PNG。

## 三、老師端流程

### 流程 A：排版

1. 老師開啟首頁。
2. 老師上傳圖片或 PDF。
3. 系統驗證檔案格式與大小。
4. 老師新增簽名欄。
5. 老師拖曳位置、調整寬高。
6. 老師確認版面。

### 流程 B：分享

1. 老師點選「產生家長連結」。
2. 系統將以下資料編碼進網址：
   - 版本號
   - 標題
   - 文件比例資訊
   - 簽名欄座標
   - 簽名欄尺寸
   - 簽名欄標籤
3. 老師複製網址。
4. 老師把網址與同意書原檔一併傳給家長。

## 四、家長端流程

### 流程 C：載入排版

1. 家長開啟老師提供的網址。
2. 系統解析網址中的排版資料。
3. 家長端顯示預設簽名欄位置。
4. 家長看到提示，要求先貼上或上傳同意書。

### 流程 D：載入同意書

1. 家長可上傳圖片。
2. 家長可上傳 PDF。
3. 家長可在支援的裝置上直接貼上圖片。
4. 系統檢查文件比例是否和老師排版時接近。
5. 若比例差異太大，系統顯示偏移警告。

### 流程 E：簽名與下載

1. 家長點選簽名欄。
2. 家長在手寫板簽名。
3. 家長點選「套用簽名」。
4. 系統將簽名放入選定欄位。
5. 家長點選「下載完成圖片」。
6. 系統輸出 PNG。

## 五、例外處理

### 情境 1：老師未先上傳同意書

系統阻止產生家長連結。

### 情境 2：老師沒有設定簽名欄

系統阻止產生家長連結。

### 情境 3：家長只開啟連結但未載入同意書

系統阻止套用簽名與下載 PNG。

### 情境 4：家長使用了不同版面的文件

系統提示：

```text
版面比例和老師排版不同，簽名欄可能偏移。
```

### 情境 5：貼上內容不是圖片

系統忽略該貼上資料，或要求改用上傳。

## 六、第二版完成定義

1. 不需要任何 Google 帳號設定即可使用。
2. 老師可以產生排版連結。
3. 家長可以使用該連結完成簽名。
4. 家長可以得到 PNG 成果圖。
5. 主要流程可在手機上操作。
