# 線上補簽同意書操作流程

本文件描述老師、家長與系統在補簽流程中的操作與資料流。此版本為開發前確認稿，確認後才開始 coding。

## 一、系統前置設定

由系統設計者完成：

1. 建立一個 Google Drive 暫存資料夾。
2. 提供該資料夾的 Google Drive folder ID。
3. 建立 Google Service Account。
4. 將暫存資料夾分享給 Service Account，給予可寫入與刪除權限。
5. 將 Drive folder ID 與 Service Account 金鑰設定到後端環境變數。
6. 設定暫存案件過期時間，第一版為 4 小時。

## 二、跨裝置流程原則

老師與家長會在不同裝置完成流程。老師可能在學校電腦或手機建立同意書，家長可能在另一支手機、平板或家中電腦開啟連結簽名。

因此系統必須符合：

1. 老師端產生的家長連結不可依賴老師裝置的 localStorage、sessionStorage 或瀏覽器暫存。
2. 家長連結應包含案件 ID 或一次性 token。
3. 家長端開啟連結後，必須可由後端依案件 ID 讀取同意書檔案與簽名欄位設定。
4. 同意書檔案存放於系統設計者提供的 Google Drive 暫存資料夾。
5. 簽名欄位座標與案件狀態需暫存在可跨裝置讀取的位置。
6. 案件完成或過期後，案件 ID 或 token 必須失效。

## 三、手機操作原則

家長端需優先支援手機瀏覽器，因家長最可能直接用手機開啟老師傳來的連結。

1. 家長可在不同手機開啟簽名連結。
2. 手寫簽名板需支援手機觸控。
3. 同意書預覽需適應手機寬度。
4. 簽名欄位置需依同意書比例縮放，避免手機畫面尺寸不同造成位置偏移。
5. 完成圖片需可在手機下載。
6. 若手機瀏覽器無法直接下載，需顯示完成圖片，讓家長可長按保存。

## 四、老師操作流程

### 流程 A：建立同意書

1. 老師開啟老師端頁面。
2. 老師建立一份同意書，可使用以下方式：
   - 複製貼上文字內容。
   - 上傳圖片同意書。
   - 上傳 PDF 同意書。
   - 上傳後由系統暫存到設計者提供的 Google Drive 資料夾。
3. 系統檢查檔案大小。
4. 若檔案超過 5MB，系統拒絕上傳並顯示提示。
5. 若檔案符合限制，系統建立暫存案件。

### 流程 B：調整簽名欄位

1. 老師在同意書預覽畫面新增簽名欄。
2. 老師拖曳簽名欄調整位置。
3. 老師調整簽名欄寬度與高度。
4. 系統記錄簽名欄座標與大小。
5. 老師可下載空白簽名欄預覽圖，確認家長尚未簽名前的版面。
6. 老師確認排版。

### 流程 C：送出給家長

1. 老師按下產生家長連結。
2. 系統將以下資料暫存：
   - 同意書檔案 ID。
   - 同意書標題。
   - 簽名欄位座標。
   - 案件 ID 或一次性 token。
   - 暫存案件到期時間。
3. 系統產生家長簽名連結。
4. 第一版一份連結只對應一位家長。
5. 老師將連結傳給家長。

## 五、家長操作流程

### 流程 D：開啟同意書

1. 家長取得老師提供的連結。
2. 家長可在另一台裝置或另一支手機不需登入即可開啟頁面。
3. 系統依案件 ID 讀取暫存同意書與簽名欄位設定。
4. 若案件不存在或已過期，系統顯示連結失效。

### 流程 E：手寫簽名

1. 家長在簽名欄或手寫板上簽名。
2. 系統將簽名套用到指定簽名欄。
3. 家長可清除並重簽。
4. 家長確認簽名。

### 流程 F：下載完成圖片

1. 家長按下下載完成圖片。
2. 系統將同意書與簽名合成為一張圖片。
3. 圖片內容必須包含：
   - 完整同意書內容或底圖。
   - 家長手寫簽名。
   - 老師設定的簽名位置。
4. 家長下載 PNG 格式圖片。
5. 系統將案件標記為完成。
6. 系統刪除該案件在 Google Drive 的暫存檔案與暫存設定。

## 六、系統內部流程

### 流程 G：暫存同意書

1. 接收老師上傳檔案。
2. 檢查檔案大小是否小於等於 5MB。
3. 檢查檔案格式是否為圖片或 PDF。
4. 寫入設計者提供的 Google Drive 暫存資料夾。
5. 建立案件 ID。
6. 儲存案件設定。
7. 回傳老師端預覽與後續編輯狀態。

### 流程 H：讀取家長簽名頁

1. 依家長連結中的案件 ID 查詢案件設定。
2. 檢查案件是否存在。
3. 檢查案件是否過期。
4. 從 Google Drive 取得同意書檔案。
5. 將同意書與簽名欄位設定回傳前端。
6. 若案件已完成，系統顯示連結已使用或已失效。

### 流程 I：完成後清除

1. 家長完成下載後，前端通知後端案件已完成。
2. 後端刪除 Google Drive 暫存檔案。
3. 後端刪除案件設定。
4. 後端回傳清除成功。

### 流程 J：過期自動清除

1. 系統定期掃描暫存案件。
2. 找出超過保留時間但尚未完成的案件。
3. 刪除 Google Drive 暫存檔案。
4. 刪除案件設定。
5. 記錄清除結果。

## 七、例外處理

### 情境 1：檔案超過 5MB

系統顯示：

```text
檔案超過 5MB，請壓縮或改用較小的同意書檔案。
```

### 情境 2：Google Drive 暫存失敗

系統顯示：

```text
同意書暫時無法上傳，請稍後再試。
```

後端需記錄錯誤原因，方便系統設計者檢查 Drive 權限或 API 設定。

### 情境 3：家長連結失效

系統顯示：

```text
此簽名連結不存在或已過期，請向老師索取新的連結。
```

### 情境 4：完成圖片產生失敗

系統顯示：

```text
圖片產生失敗，請重新整理後再試一次。
```

## 八、驗收標準

1. 老師端可建立同意書案件。
2. 老師端可拒絕超過 5MB 的檔案。
3. 老師端可把同意書暫存到指定 Google Drive folder ID。
4. 老師端可新增、拖曳、調整簽名欄。
5. 老師端可下載空白簽名欄預覽圖。
6. 老師端可產生家長連結。
7. 第一版一份連結只對應一位家長。
8. 家長端可在不同裝置與不同手機不需登入即可開啟連結。
9. 家長端可顯示同意書與簽名欄。
10. 家長端可手寫簽名並套用到文件。
11. 家長端可下載 PNG 格式且包含同意書與簽名的圖片。
12. 家長下載完成後，系統會自動移除暫存檔案。
13. 未完成案件會在 4 小時後自動移除。
14. 關閉老師端瀏覽器後，家長連結仍可正常開啟未過期案件。
15. 家長端在手機瀏覽器可觸控簽名並取得完成圖片。

## 九、已確認決策

1. 暫存案件保留時間設定為 4 小時。
2. 第一版同意書格式支援圖片與 PDF。
3. 完成圖片格式使用 PNG。
4. 老師端需要下載空白簽名欄預覽圖。
5. 第一版先做一份連結一位家長。
